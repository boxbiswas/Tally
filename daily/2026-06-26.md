Summary
-------
- Continued from the 2026-06-25 handoff, focusing entirely on UI/UX refinements, keyboard-driven navigation polish, and improving the report/print invoice aesthetics.
- Streamlined the global shortcut system and integrated keyboard navigation deeper into the reporting and data tables.
- Overhauled the Print Invoice format to align closer with traditional ERP/Tally outputs.
- Refactored backend reporting queries for more robust data inclusion and aggregations.
- Enforced security scoping by companyId across all backend controllers.
- Separated form fields into reusable components and fixed minor UX bugs (e.g., masking 0s in number inputs).

Backend
-------
- Refactored `dashboardController.js` to utilize Prisma's native `.count()` and `.aggregate()` methods for cleaner and more performant metric calculations.
- Updated `reportController.js` to fetch comprehensive relational data, ensuring sales vouchers include customer details and purchase vouchers include supplier details using Prisma's `include` clauses.
- Verified that all report routes in `reportRoutes.js` are securely protected behind the existing `authMiddleware`.
- Scoped all primary controllers (`ledgerController`, `purchaseController`, `reportController`, `salesController`, `stockController`) explicitly to `companyId` to prevent cross-company data leakage.

Frontend
--------
- Overhauled the **Print Invoice** page (`PrintInvoice.jsx`) to feature a clean, professional ERP-style layout with improved typography, spacing, and a standardized tabular structure for line items.
- Enhanced the **Reports** page (`Reports.jsx`) with deep keyboard navigation support:
  - Added row-level selection state and visual highlighting (e.g., blue background for selected rows).
  - Implemented keyboard event listeners (Up/Down arrows, Enter) so users can navigate reports without a mouse.
  - Allowed direct navigation to the print invoice view by pressing `Enter` on a selected sales voucher row.
  - Added loading states and empty state fallbacks for tables.
- Refined the keyboard shortcut and form navigation custom hooks (`useFormNavigation.js`, `useShortcuts.js`). Consolidated the shortcut logic by entirely removing the redundant `useGlobalShortcuts.js` hook.
- Updated the main application layouts (`MainLayout.jsx`, `Sidebar.jsx`, `ShortcutSidebar.jsx`) for a cleaner visual hierarchy and better presentation of available F-key shortcuts to the user.
- Wrapped the app in a `<ShortcutProvider>` for more robust global shortcut handling.
- Separated form fields for `ItemForm` and `LedgerForm` into dedicated components (`ItemFormFields.jsx`, `LedgerFormFields.jsx`).
- Refined `useFormNavigation` for better `SELECT` dropdown and `INPUT` escape logic.
- Updated `ItemForm` to display live stock quantity natively.
- Added `.rejected` logout toast handling in `authSlice.js`.
- Refined `CompanyForm.jsx` UI and UX: Combined the "Back to Companies" button and "ESC" shortcut indicator into a unified, styled box design.
- Integrated `CompanyForm.jsx` with the global `useFormNavigation` hook by updating its root container to `<main>`, restoring consistent, Tally-like arrow key and Enter key navigation across form fields.
